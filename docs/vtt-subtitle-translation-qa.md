# VTT 字幕翻譯 QA 筆記

這份筆記用來檢查「英文 VTT → 繁體中文字幕」的交付品質。重點是不要相信模型自稱完成；字幕任務應該用規則先擋掉格式錯誤，再用抽樣審稿處理語氣和笑點。

## QA 目標

- VTT 結構不壞：標頭、時間碼、cue 數、空行分隔都要保留。
- TSV 對齊不錯位：每個 ID 都有、只對應一條譯文。
- 譯文不是只有「意思正確」，還要像台灣字幕。
- 人物語氣、現場節奏、笑點和前後呼應不能被翻平。

## 機械檢查

### 1. ID 完整性

- 每批輸出必須涵蓋該批所有目標 ID。
- ID 必須是 4 位數。
- 每行必須是 `ID<TAB>譯文`。
- 不可輸出前文參考的 ID。
- 重複 ID 要警告；若保留「最後一版」，要確認這不是模型錯位造成的。
- 缺 ID、外部 ID、無法解析的行都要直接失敗。

### 2. TSV 格式

- 每行只能有一個欄位分隔 TAB。
- 譯文內不能有 TAB。
- 不可有 Markdown code fence、項目符號、前言、結語。
- 不接受 `0001: English -> 中文` 這類格式；若腳本做容錯，也要記錄為格式瑕疵。

### 3. VTT 結構

- 輸出必須以 `WEBVTT` 開頭。
- 來源與輸出的時間碼行數必須一致。
- 來源與輸出的每一行時間碼必須逐行完全相同。
- 輸出 block 數應等於 cue 數加 1 個 header block。
- `Language` 可改成 `zh-TW`，其他非必要 metadata 不要亂改。

### 4. 英文殘留

要抓完整英文句子殘留，但允許合理專名與縮寫。

可允許：

- 人名、團名、品牌、節目名。
- MBTI、API、CEO、YouTube、BLACKPINK 這類固定縮寫或產品名。
- 原文刻意保留的外語詞。

要攔下：

- 完整英文句子。
- 明顯沒有翻完的英文片段。
- 同一 cue 同時出現英文草稿和中文成品。

### 5. 高風險錯字與斷詞

模型常會產生「看起來像中文，但其實是錯字或錯切」的問題。QA 要特別掃：

- 同音或近形錯字，例如「於勢」應為「於是」。
- 錯誤斷詞，例如「讀稿機會」可能其實是「讀稿機 會...」或應重寫成「讀稿機會把我們拉回來」。
- 人名、地名、品牌名拼錯。
- 繁簡混用。
- 句尾標點大量不一致。

## 人工抽樣審稿

機械檢查只能保證格式，不能保證字幕好看。每次完整輸出後，至少抽樣看：

- 開頭 30 cue：確認節目語氣和譯名方向。
- 每個批次邊界前後各 20 cue：確認上下文沒有斷。
- 笑點密集段：確認吐槽、反差、回扣沒有被翻平。
- 專名密集段：確認人名、品牌、節目名一致。
- 結尾 30 cue：確認語氣沒有漂移。

## 風格審查

### 台灣字幕感

- 台灣觀眾會自然讀懂。
- 句子短、清楚、有口語節奏。
- 不過度正式，不像客服或新聞稿。
- 不硬翻英文片語；需要時改成台灣人會說的講法。

### 人物語氣

- 主持人、來賓、旁白不能全部同一種聲音。
- 誇張、捧人、自嘲、冷幽默、尷尬停頓，都要在中文裡有對應的節奏。
- 不要把內斂的人翻得太活潑，也不要把吐槽翻成正式陳述。

### 笑點節奏

- 笑點不是只翻意思，還要保留前後鋪陳和反差。
- 前一 cue 埋的梗，後一 cue 回收時要看得出來。
- 直譯會讓笑點死掉時，可以自然改寫，但不能加新梗。

## 字幕翻譯 case 檢查庫

審稿時不要只問「有沒有翻錯」。也要逐段掃以下 case，這些是英文訪談字幕翻成台灣繁中時最常出問題的地方。

### 1. 跨 cue 斷句

- 連續 cue 是否其實是同一句？
- 中文是否被切成「我想。」「然後。」「如果你思考它。」這種假完整句？
- 每個 cue 是否仍能接上前後語氣？

### 2. 口頭禪與填充詞

- `I mean`、`like`、`you know`、`kind of` 是否逐字硬翻？
- 該省略的口頭墊片是否有省略？
- 重要的猶豫、尷尬、心虛是否仍有保留？

### 3. 代詞與省主詞

- `it`、`that`、`this`、`you` 是否翻成不自然的「它／那個／這個／你」？
- 中文是否可以直接省主詞？
- 是否需要根據上下文補出「那時」「這件事」「這樣講」？

### 4. 假朋友與英文常用句

- `actually` 是否被濫翻成「實際上」？
- `I'm good` 是否被誤翻成「我很好」？
- `That's fair` 是否被誤翻成「那是公平的」？
- `I feel like` 是否被誤翻成「我感覺像是」？

### 5. 角色聲音

- 主持人、來賓、專家、朋友是否聽起來都是同一個人？
- 誇張捧人、自嘲、冷幽默、正式說明是否有不同中文聲音？
- 語氣是否符合關係，而不是突然太尊敬或太親密？

### 6. 反諷與吐槽

- `Sure, very helpful.` 是否被翻成真心稱讚？
- `Good luck with that.` 是否保留了微妙的吐槽感？
- `No pressure.` 是真的沒壓力，還是在反諷？

### 7. Callback 與固定梗

- 前面建立的比喻、百分比、稱呼、綽號是否後面一致使用？
- 後面回扣前文時，觀眾是否看得出來？
- 是否因分批翻譯導致同一個梗前後不同譯法？

### 8. 文化梗與慣用語

- `elephant in the room`、`walking on eggshells`、`main character energy` 是否照字面翻？
- 原文比喻是否需要保留，還是應翻成觀眾秒懂的意思？
- 若文化資訊不可省，是否用最短方式處理，而不是加長譯註？

### 9. 閱讀速度與字幕壓縮

- 中文是否太長，觀眾在畫面時間內讀不完？
- 英文修飾語是否全塞進中文，造成喘不過氣？
- 是否能保留重點並縮短，而不是逐字搬運？

### 10. 專名與稱謂

- 人名、團名、品牌、節目名是否一致？
- 稱謂是否符合語境，例如 senior 多半是「前輩」，不是「年長者」？
- YouTube Shorts、MBTI、CEO 等固定詞是否被亂翻？

### 11. 語氣強弱與不確定性

- `maybe`、`probably`、`I guess` 是否被翻得過度肯定？
- 原本是試探、保留、心虛，中文是否變成直接定論？
- 該用「吧」「大概」「可能」時是否自然，不要硬塞。

### 12. 反問、echo、接話

- 「第一天就？」「真的？」「等一下」這類互動節奏是否有保留？
- 英文問句是否被翻成僵硬完整句？
- 重複對方用詞時，中文是否也保留接話感？

### 13. 情緒細節

- 尷尬、逞強、裝沒事、受寵若驚、被拆台是否被翻平？
- 是否有把情緒加過頭，變成原文沒有的 drama？
- 短句是否仍然短，不要擅自補成長篇內心戲。

### 14. 台灣口語的分寸

- 是否每句都加「啦」「欸」「耶」「超」，導致所有人變同一種台味？
- 正式、真誠、低調的段落是否被改得太鬧？
- 口語化是否服務語氣，而不是把原文 tone 改掉？

### 15. 二手英文字幕

- 英文字幕是否本來就是從韓文、日文等語言翻過來的？
- `close`、`promise`、`senior`、`aegyo` 這類詞是否需要還原成東亞綜藝語境？
- 是否被英文中介語帶偏，翻出字面正確但場景錯誤的中文？

### 16. 多 speaker 同 cue

- 破折號前後是否是不同人說話？
- 「真的假的？」「對」「不是不是」這類接話節奏是否保留？
- 是否把短來短往翻成兩句正式完整句，導致節奏變慢？

### 17. 反應詞與擬聲詞

- `Whoa`、`Aww`、`Oops`、`Ew`、`Hmm` 是否依功能處理？
- 反應詞是驚訝、心疼、嫌棄、尷尬，還是在拖時間？
- 是否因逐字翻譯讓反應變得很假？

### 18. 強烈語氣與髒話強度

- 英文髒話是否依節目 tone 調整，不要太髒也不要洗太乾淨？
- `what the hell`、`freaking`、`lost it` 是否翻成自然台灣語氣？
- 情緒強度是否保留，但沒有新增原文沒有的攻擊性？

## 建議流程

### 第一段：忠實翻譯

先確保意思、專名、數字、格式都對。這一段不要追求花俏。

### 第二段：台灣口語潤稿

把翻譯腔修掉，讓每句像自然中文字幕。重點是順口、短、清楚。

### 第三段：上下文與人物語氣 QA

回頭看前後 cue，修正人物聲音、笑點節奏、批次邊界斷裂和稱呼不一致。

## 驗收標準

可以交付：

- 所有機械檢查通過。
- 沒有完整英文句子殘留。
- 抽樣段落讀起來像台灣字幕。
- 批次邊界沒有明顯斷裂。
- 主要人物語氣可分辨。

需要重跑或重修：

- 缺 ID、錯 ID、時間碼不一致。
- 大量英文殘留。
- 明顯翻譯腔集中出現。
- 笑點被翻成平鋪直敘。
- 同一人物前後稱呼或語氣明顯漂移。
